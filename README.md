# dynamic_array

# Динамічний масив - звіт

## Реалізація
- Динамічний масив на ctypes
- Збільшує ємність при додаванні
- Підрахунок:
  - кількість перевиділень памʼяті
  - фінальна ємність
  - невикористані елементи

**Стратегії зростання**
1. ×2 — швидко, багато памʼяті
2. 1.5 — збалансовано (номер 5)
3. variable — повільно на початку, часті resize пізніше

## Результати
- 8, 10000 - Final capacity: 11623
Resize count: 18
- 32, 100000 - Final capacity: 106204
Resize count: 20
- 128, 1000000 - Final capacity: 1436239
Resize count: 23

## Висновки
- ×2 — найшвидше, памʼять витрачає багато  
- 1.5 — збалансовано, оптимально  
- variable — часто resize, повільніше
